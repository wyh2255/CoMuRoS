## 数据流

```
用户输入 (GUI 文本)
  → [新路径] HTTP POST /tasks → Coordinator (RouterAgent LLM 分解)
  → A2A tasks/send → Worker Mini-Agent (Tool Calling)
  → ROS service → Gazebo 仿真
  → Worker step_callback → /chat/output → GUI 显示

  → [旧路径, USE_A2A_COORDINATOR=0] /chat/input
  → task_manager.py (LLM 解析) → /task_manager/tasks_json
  → Robot LLM 节点 (exec 生成的 Python 代码)
```

## 核心组件

### 1. A2AWorkerNode (CoMuRoS)

**文件:** `CoMuRoS/robot_llm/robot_llm/a2a_worker_node.py`

所有 robot Worker 的基类，提供:

| 功能 | 说明 |
|------|------|
| **ROS pub/sub** | /chat/task_status, /robot_states, {name}_task_status |
| **A2A HTTP Server** | 后台线程运行 uvicorn, Mini-Agent 执行器 |
| **WebSocket 注册** | Coordinator WS 注册 + 30s 心跳 (自动重连) |
| **聊天历史** | 注入 system_prompt, 文件持久化, 线程安全锁 |
| **状态管理** | robot_states 合并发布, task_status 发布 |
| **任务取消** | check_cancelled() / TaskCancelledException |
| **子系统钩子** | `_get_tools()`, `_get_capabilities()`, `_get_system_prompt()`, `_get_model()` |

**路径解析:** `MY_A2A_PATH` / `MINI_AGENT_PATH` 环境变量 → fallback 相对路径

### 2. MiniAgentAdapter (my_a2a)

**文件:** `my_a2a/src/openharness_a2a/worker/mini_agent_adapter.py`

桥接 Mini-Agent Agent.run() 到 A2A EventQueue:
- `execute()` — 接收 A2A 任务, 调用 Agent.run(), 推送事件
- `cancel()` — 取消 Agent 执行
- `_build_agent()` — 懒初始化 Agent (支持 Config 和 fallback)
- `_on_step_event()` — 转换 Mini-Agent 事件为 A2A 事件

**新增参数:** `extra_tools`, `system_prompt`, `step_callback`

### 3. CoordinatorServer (my_a2a)

**文件:** `my_a2a/src/openharness_a2a/coordinator/server.py`

FastAPI 主服务, REST :8080 + A2A :8081 + WebSocket:
- `POST /tasks` — 创建任务, 返回 task_id
- `GET /tasks/{task_id}` — 轮询任务状态
- `WS /ws/worker/{worker_id}` — Worker 注册通道
- `config_path` 参数 — 支持自定义 AgentRegistry 配置

### 4. RouterAgent (my_a2a)

**文件:** `my_a2a/src/openharness_a2a/coordinator/router.py`

LLM 驱动的智能路由:
- `ROUTER_SYSTEM_PROMPT` — 餐厅环境描述 (3 stalls, 4 tables, 1 sink)
- `_build_system_prompt()` — 注入动态 Worker 状态 (online/offline, busy/idle)
- `route()` — 调用 LLM + 解析 JSON 路由决策
- `push_task()` — 推送子任务到 Worker A2A 端点

**输出格式:**
```json
{
  "subtasks": [{"agent_id": "cleaning-bot", "prompt": "..."}],
  "execution_order": "sequential|parallel|single",
  "reasoning": "..."
}
```

### 5. Robot Tools (CoMuRoS)

每个 robot 注册为 Mini-Agent Tool:

| Tool | Robot | 参数 | ROS 后端 |
|------|-------|------|----------|
| CleanTool | cleaning_bot (:8091) | 无参数 | GotoPoseHolonomic |
| DeliverFoodTool | delivery_bot (:8092) | stall_number, table_number | GotoPoseHolonomic + teleport |
| ClearTableTool | delivery_bot | table_number, food_name | GotoPoseHolonomic + teleport |
| HoverTool | drone (:8093) | x, y, z, yaw_deg | GotoPoseDrone |
| DescribeSceneTool | drone | prompt (VLM 查询) | GPT-4.1 Vision |
| PickObjectTool | robot_arm (:8094) | object_name | StartPick service |

### 6. GUI 集成

**文件:** `CoMuRoS/chatty/chatty/chat_gui.py`
- 新增 `httpx` 依赖
- `coordinator_url` ROS 参数 (默认 `http://localhost:8080`)
- `send_message()` → ROS `/chat/input` + HTTP POST `/tasks`
- `_send_to_coordinator()` → 后台线程轮询 (1s × 60s)
- `on_output_direct()` → 直接渲染结果到 GUI

### 7. task_manager 切换

**文件:** `CoMuRoS/chatty/chatty/task_manager.py`
- `USE_A2A_COORDINATOR=1` → 跳过启动 (task_manager 禁用)
- `chat_system.launch.py` 默认设 `USE_A2A_COORDINATOR=1`

## 启动步骤 (A2A 模式)

```bash
# Terminal 1: 仿真 + robots
ros2 launch multi_robot multi_robot.launch.py
ros2 launch robot_llm robot_llms.launch.py
ros2 launch robot_llm robot_services.launch.py

# Terminal 2: GUI + Coordinator
ros2 launch chatty chat_system.launch.py
cd my_a2a && openharness-a2a
```
